Loading Your Data

This tool is built to run on a single static HTML file with no backend. To plug in real UCSF Health data, populate the LEADERSHIP_DATA JavaScript object at the bottom of index.html. The shape below matches the existing Physician_Leader_Cost_Center_Map_Master_16.xlsx structure.

1 · Top-level structure

// Sits inline in index.html — replace the empty arrays
const LEADERSHIP_DATA = {
  svp:  { name: "Inga Lennes", ccCount: 655 },
  cmo:  { name: "John Fangman", ccCount: 343, aemdCount: 8 },
  aemds: [ /* see schema below */ ]
};

2 · AEMD record

4 · Where the data comes from

  • AEMD names + CC counts → from Physician_Leader_Cost_Center_Map_Master_16.xlsx
  • MD assignments + dyads → from the same map plus Med_Director__DATA_TABLE_1.xlsx
  • Cost center rows → ID, name, division, admin director, manager columns from the master map
  • Rubric metrics (optional) → only populate once a role has been scored; the cascade view will then show a teal score pill next to the MD name

5 · Conversion workflow

Easiest path: open the Excel file, save the relevant sheet as CSV, then convert to JSON with any free converter (or a one-line Python script with pandas). Paste the result into LEADERSHIP_DATA.aemds. The tool will render automatically — no build step.

6 · GitHub deployment

  • Create a new repo, drop in index.html and README.md
  • Settings → Pages → Source: main branch, /root
  • Your tool will be live at https://<username>.github.io/<repo>/
  • No build pipeline, no Node dependencies — pure static HTML/CSS/JS

7 · Privacy note

Because everything renders client-side and saved rubric scores live in localStorage, no data leaves the user's browser. If you need to share scores across the workgroup, use the Export CSV button on the rubric tab.
